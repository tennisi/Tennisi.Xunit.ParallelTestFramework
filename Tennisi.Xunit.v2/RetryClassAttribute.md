# RetryClassAttribute class

Represents a custom xUnit attribute to specify that all test methods in the class (both facts and theories) should be retried a specified number of times.

```csharp
[AttributeUsage(AttributeTargets.Class)]
public sealed class RetryClassAttribute : Attribute
```

## Public Members

| name | description |
| --- | --- |
| [RetryClassAttribute](RetryClassAttribute/RetryClassAttribute.md)(…) | Initializes a new instance of the [`RetryClassAttribute`](./RetryClassAttribute.md) class with the specified retry count. |
| [RetryCount](RetryClassAttribute/RetryCount.md) { get; } | Gets the number of times the test methods in the class should be retried. |

## Remarks

This attribute is useful for test classes where tests may occasionally fail due to transient issues such as network delays, timeouts, or other external dependencies. When applied, each test method in the class, including both FactAttribute and TheoryAttribute methods, will be retried up to the specified [`RetryCount`](./RetryClassAttribute/RetryCount.md) value upon failure.

## Examples

```csharp
[RetryClass(retryCount: 3)]
public class MyTests
{
    [Fact]
    public void TestMethod1()
    {
        // Test code for TestMethod1
    }
    
    [Theory]
    [InlineData(1)]
    [InlineData(2)]
    public void TestMethod2(int input)
    {
        // Test code for TestMethod2
    }
}
```
